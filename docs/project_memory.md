- **存檔系統**: JSON主導 + PlayerPrefs備援的雙重保險機制

- **多角色支援**: 未來需考慮多女配角系統，需要角色偵測機制

### ⚠️ 注意事項
- 多角色系統擴展性考量 - 不僅是女主角，還有女配角的劇情分支
- JSON數據結構需要兼容現有PlayerPrefs系統
- 保持"原始人"的簡潔設計理念 - 避免過度工程
- 🚨 **AI協作限制 (2025-08-23重大發現)**: AI記憶體限制導致邊讀邊忘，無法完全依照MD文件規格開發
- 🎯 **代碼符合度問題**: AI生成代碼雖然語法正確，但容易偏離專案具體需求和規格
- 🧠 **遊戲企劃複雜性**: 遊戲機制抽象且複雜，需要開發者親自深度規劃，不能完全外包給AI
- 🤖 **過度依賴AI風險**: "AI萬能主義"是不現實的，開發者仍需主導架構和業務邏輯設計
- 📋 **"有規格的限制嘴砲"**: 即使有詳細MD文件，仍無法替代實際開發經驗和專案深度理解
- ❌ **重製計劃失敗教訓**: 全新腳本系統過於龐大，脫離現有專案基礎，決定改為疊加式開發
- 🔄 **新協作模式**: 開發者手寫中文註解架構 + AI補齊C#語法，發揮各自優勢
- 📚 **NEW腳本系統價值**: 雖然不採用，但作為優質參考資料，部分代碼可沿用
- 🎮 **回歸現實策略**: 在舊場景GamePlayScene基礎上繼續疊加更新，避免重複造輪子

## 🎮 遊戲系統概覽

### 🎭 核心遊戲模式

#### 📜 劇情模式
- **特色**: 靜態立繪 + CG畫面
- **互動**: 對話文字進行、選擇分支
- **角色**: 使用 ActorManagerDrama 控制立繪表情
- **展示**: 見 `實機畫面/劇情模式*.png`

#### 💖 養成模式
- **特色**: 酒吧場景 + Yuka Spine動畫
- **互動**: 點擊角色、對話、外出系統
- **角色**: 1位主角(Spine) + 多位配角(靜態)
- **展示**: 見 `實機畫面/養成模式*.png`

#### ⚙️ 系統設定
- **功能**: BGM/音效/語音音量控制
- **界面**: 視窗模式、語言切換
- **存檔**: 存檔/讀檔系統
- **展示**: 見 `實機畫面/設定.png`

### 🎯 技術架構特點

#### 🎨 演員控制系統
n#### 🚨 混合架構系統 (2025-08-21重大發現)- **WorldSpace + Canvas混合**: Yuka立繪在Canvas破圖，強制使用混合架構- **角色顯示**: WorldSpace SkeletonAnimation，避免Canvas渲染問題- **背景環境**: WorldSpace物件，保持3D空間效果- **UI互動層**: Canvas透明Button覆蓋，實現點擊檢測- **純UI元素**: 選單、對話框、設定等使用Canvas實現- **座標轉換**: WorldSpace↔ScreenSpace座標系統轉換
- **Yuka (主角)**: Spine動畫，四軌道表情控制(眉毛、眼睛、嘴巴、臉部)
- **其他角色**: 靜態立繪，圖片切換系統
- **統一接口**: 所有角色使用相同的 `IActorController` 接口
- **資源優化**: 5位女角使用統一系統，僅修改前綴名稱

#### 📦 數據流系統
- **JSON 驅動**: 對話、設定、角色數據全部使用 JSON
- **GameManager 中心化**: 所有系統經由 GameManagerTest 統一調度
- **EventBus 通訊**: 模組間使用事件系統通訊，保持低耦合

#### 🛠️ 開發理念
- **小團隊優化**: 兩人團隊，資源有限，重視實用性
- **核心功能專注**: 主要功能是表情系統和互動對話
- **模組化設計**: 高度可擴展，但保持簡潔
- **成人內容重點**: 此模式功能較為豐富，其他模式相對簡潔

## 📞 協助指南

### 🤝 如何與 Claude 協作 (2025-08-23 新模式)
1. **架構主導權**: 開發者負責業務邏輯設計，提供中文註解架構代碼
2. **語法輔助角色**: Claude 負責將中文註解轉換為完整的 C# Unity 語法
3. **參考資料利用**: 善用之前生成的優質代碼作為開發參考
4. **疊加式開發**: 在現有舊場景基礎上逐步添加功能，避免大規模重構
5. **更新記錄**: 完成任務後更新 `docs/ProjectLogs/`

### 💡 協作流程範例
```
開發者提供:
start() { 開始遊戲動畫(); 播放遊戲音效(); }
void 開始遊戲動畫(){ ui開場動畫(); 角色開場動畫(); }

Claude 轉換為:
using UnityEngine;
public class GameStartManager : MonoBehaviour
{
    void Start() { 開始遊戲動畫(); 播放遊戲音效(); }
    void 開始遊戲動畫() { UI開場動畫("UI_FadeIn", 1.2f); 角色開場動畫("Hero01", "IdleToReady"); }
}
```

### 📋 常見任務類型
- **新功能開發** - 新增遊戲機制或角色互動
- **程式碼重構** - 模組化和性能優化
- **文件更新** - 保持文檔與程式碼同步
- **架構優化** - 系統設計和效能提升
- **問題排除** - Bug修復和系統調試
